Homework 1: Tasks for ba257

Your personal task is to develop a system for gyms and their trainers.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Trainer. With the following private fields:

  1. String name
  2. String skills
  3. String expertise
  4. int duration
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Gym. With the following private fields:

  1. int id
  2. String classes
  3. String website
  4. List<Trainer> trainers
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Gym> gyms in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one gym with two trainers. [4 marks]

Create controller co2123.hw1.controller.GymController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/gyms" should show view gyms/list. [2 marks]
    • Add the list of all gyms from co2123.hw1.Hw1Application to the model as attribute gyms. [2 marks]
    • Create the corresponding JSP to list the classes and website of each gym followed by a link to URL "/trainers?gym={id of gym}", e.g., "/trainers?gym=3". [4 marks]
  2. Requests for URL "/newGym" should show view gyms/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="gym" and action="/addGym" and inputs for all fields of class Gym except for trainers. [5 marks]
  3. POST requests for URL "/addGym" should add a new gym to the list of all gyms in co2123.hw1.Hw1Application and redirect to /gyms. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.TrainerController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/trainers" should show view trainers/list. [2 marks]
    • Based on request parameter gym determine the corresponding gym from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all trainers of that gym to the model as attribute trainers. [2 marks]
    • Create the corresponding JSP for the view to list the name, skills, expertise, and duration of each trainer. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newTrainer?gym={id of gym whose trainers are listed}", e.g., "/newTrainer?gym=3". [4 marks]
  2. Requests for URL "/newTrainer" should show view trainers/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="trainer" and action="/addTrainer" and inputs for all fields of class Trainer. [6 marks]
    • Also add the request parameter gym with the ID of the current gym. [4 marks]
  3. POST requests for URL "/addTrainer" should add a new trainer to the list of trainers of the gym identified by request parameter gym and then redirect to /gyms. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Gym and make it available in controller GymController. [4 marks]

When adding a new gym validate that

  1. The value of field id is unique within the list Hw1Application.gyms. [3 marks]
  2. The value of field classes is not empty. [2 marks]
  3. The value of field website is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Trainer and make it available in controller TrainerController. [4 marks]

When adding a new trainer validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field skills is not empty. [2 marks]
  3. The value of field expertise is either "Tai Chi", "Pilates", or "Yoga". [3 marks]
  4. The value of field duration is between (including) 45 and 60. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
